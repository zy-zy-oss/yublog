hook ​

React Hook 是 React 自 16.8 版本引入的一种新特性，它让函数组件（Function Component）能够使用状态（state）和生命周期（lifecycle）等功能，而这些功能原本只有类组件（Class Component）才具备。

通过 Hook，你可以更优雅地构建 React 组件，同时避免复杂的类组件逻辑和嵌套。

Hook 通俗理解 ​

可以把 Hook 想象成：

  1. 拎包入住的工具箱 ：直接给你准备好一堆功能（比如状态管理、监听副作用等），不用自己费劲设计。
  2. 打破规则的桥梁 ：在 React 里，函数组件以前是“静态的”，不能存状态，也没办法处理复杂逻辑。Hook 改变了这个规则，让函数组件也能变得动态。

  1. useEffect ：获取外部数据。用于管理副作用（如数据获取、订阅或 DOM 操作），是 React 中函数组件替代生命周期方法（如 componentDidMount、componentDidUpdate 和 componentWillUnmount）的主要工具。 例如，异步获取外部数据，对已经渲染的结果进行更新渲染，我的理解是先忽略他渲染其他的，等有结果的时候再更新。也就是每次渲染之后再执行useEffect函数

    副作用代码只在特定情况下运行，而不是每次渲染都运行。

    【又是长的】useEffect丨React的hook函数基本使用 - React18基础系列丨讲师·景水】

jsx
   import { useEffect, useState } from 'react';

   function FetchData() {
     const [data, setData] = useState(null);

     useEffect(() => {
       fetch('https://api.example.com/data')
         .then((response) => response.json())
         .then((data) => setData(data));
     }, []); // 空数组表示仅在组件挂载和卸载时运行

     return <div>{data ? JSON.stringify(data) : 'Loading...'}</div>;
   }
  1. useContext ：允许在组件树中共享状态，无需手动通过 props 逐层传递。
jsx
   import { createContext, useContext } from 'react';

   const ThemeContext = createContext('light');

   function ThemeButton() {
     const theme = useContext(ThemeContext); // 直接获取上下文中的值
     return <button style={{ background: theme === 'dark' ? 'black' : 'white' }}>Click me</button>;
   }

   function App() {
     return (
       <ThemeContext.Provider value="dark">
         <ThemeButton />
       </ThemeContext.Provider>
     );
   }
  1. useRef ：访问 DOM 节点或持久化某些值。
jsx
   import { useRef } from 'react';

   function TextInput() {
     const inputRef = useRef();

     const focusInput = () => {
       inputRef.current.focus(); // 访问 DOM 元素
     };

     return (
       <div>
         <input ref={inputRef} />
         <button onClick={focusInput}>Focus the input</button>
       </div>
     );
   }
  1. useReducer ：用于管理更复杂的状态逻辑，类似于 Redux 的 reducer。
jsx
   import { useReducer } from 'react';

   function reducer(state, action) {
     switch (action.type) {
       case 'increment':
         return { count: state.count + 1 };
       case 'decrement':
         return { count: state.count - 1 };
       default:
         throw new Error();
     }
   }

   function Counter() {
     const [state, dispatch] = useReducer(reducer, { count: 0 });

     return (
       <div>
         <p>Count: {state.count}</p>
         <button onClick={() => dispatch({ type: 'increment' })}>+</button>
         <button onClick={() => dispatch({ type: 'decrement' })}>-</button>
       </div>
     );
   }

Hook 的优点 ​

  1. 简化代码结构 ：将逻辑分块，减少类组件中的复杂代码。
  2. 避免“this”问题 ：函数组件不需要 this，减少了潜在错误。
  3. 更灵活的状态管理 ：可组合多个 Hook，清晰地管理组件逻辑。
  4. 便于复用逻辑 ：通过自定义 Hook 共享状态逻辑，而不需要高阶组件或 Render Props。

Hook 的规则 ​

  1. 只能在函数组件的最外层调用 ：不能在循环、条件语句或嵌套函数中使用 Hook。
  2. 只能在 React 函数组件或自定义 Hook 中调用 ：不要在普通的 JavaScript 函数中使用 Hook。
