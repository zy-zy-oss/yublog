useReducer 是 React 的一个 Hook，提供了状态管理的另一种方式。它类似于 Redux 的 reducer 概念，适合用于复杂状态逻辑或多个子状态的管理。

基本概念 ​

useReducer 的主要思想是通过 dispatch 分发动作，使用 reducer 函数来更新状态，而不是直接调用 setState。

jsx
const [state, dispatch] = useReducer(reducer, initialState);

参数说明： ​

  1. reducer ：一个纯函数 (state, action) => newState。
  • 它接收当前状态和一个描述动作的对象 action，并返回新的状态。
  1. initialState ：状态的初始值。
  2. （可选） init 函数 ：用于惰性初始化状态，提供一个回调函数来生成初始值。

返回值： ​

  • state ：当前状态值。
  • dispatch ：一个函数，用于分发动作（action）。

使用场景 ​

  • 状态逻辑复杂且包含多个子状态。
  • 下一个状态依赖于上一个状态。
  • 想要更好地组织状态逻辑。

基础使用 ​

以下是一个计数器的例子：

jsx
import React, { useReducer } from "react";

// 定义 reducer 函数
function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };
    case "decrement":
      return { count: state.count - 1 };
    case "reset":
      return { count: 0 };
    default:
      throw new Error("Unknown action type");
  }
}

function Counter() {
  // 使用 useReducer 初始化状态
  const [state, dispatch] = useReducer(reducer, { count: 0 });

  return (
    <div>
      <h1>Count: {state.count}</h1>
      <button onClick={() => dispatch({ type: "increment" })}>+</button>
      <button onClick={() => dispatch({ type: "decrement" })}>-</button>
      <button onClick={() => dispatch({ type: "reset" })}>Reset</button>
    </div>
  );
}

export default Counter;

运行流程： ​

  1. 初始状态为 { count: 0 }。
  2. 点击按钮时调用 dispatch，触发 reducer 函数。
  3. 根据 action.type 更新状态，React 会重新渲染组件。

复杂状态管理 ​

假如我们需要管理一个表单，useReducer 能很好地组织多个子状态。

jsx
import React, { useReducer } from "react";

// 定义 reducer 函数
function reducer(state, action) {
  switch (action.type) {
    case "updateField":
      return { ...state, [action.field]: action.value };
    case "reset":
      return { username: "", email: "" };
    default:
      throw new Error("Unknown action type");
  }
}

function Form() {
  const [state, dispatch] = useReducer(reducer, { username: "", email: "" });

  return (
    <form>
      <input
        type="text"
        value={state.username}
        onChange={(e) =>
          dispatch({ type: "updateField", field: "username", value: e.target.value })
        }
        placeholder="Username"
      />
      <input
        type="email"
        value={state.email}
        onChange={(e) =>
          dispatch({ type: "updateField", field: "email", value: e.target.value })
        }
        placeholder="Email"
      />
      <button
        type="button"
        onClick={() => dispatch({ type: "reset" })}
      >
        Reset
      </button>
      <p>Username: {state.username}</p>
      <p>Email: {state.email}</p>
    </form>
  );
}

export default Form;

运行流程： ​

  1. 用户输入时触发 updateField 动作，更新对应字段。
  2. 点击重置按钮触发 reset 动作，重置所有字段。

惰性初始化 ​

如果初始状态计算较为复杂，可以使用 init 函数进行惰性初始化。

jsx
import React, { useReducer } from "react";

function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };
    default:
      throw new Error("Unknown action type");
  }
}

function init(initialCount) {
  return { count: initialCount };
}

function Counter({ initialCount }) {
  const [state, dispatch] = useReducer(reducer, initialCount, init);

  return (
    <div>
      <h1>Count: {state.count}</h1>
      <button onClick={() => dispatch({ type: "increment" })}>+</button>
    </div>
  );
}

export default Counter;

与 useState 对比 ​

  • useState 更简单，适合管理单一状态。
  • useReducer 更适合复杂状态逻辑和多个状态。
特点useStateuseReducer
状态管理单一状态复杂状态
状态更新方式setState方法dispatch分发 action
可读性和维护性简单状态时更高复杂逻辑时更高

总结 ​

useReducer 是 useState 的增强版，适合复杂状态逻辑。关键点：

  1. 状态更新通过 dispatch 分发动作。
  2. 状态逻辑集中在 reducer 函数中。
  3. 在大型应用中，useReducer 能提升代码的组织性和可维护性。
