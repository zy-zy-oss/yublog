useRef详解 ​

useRef 是 React 提供的一个 Hook，用于在函数组件中创建对 DOM 元素或组件实例的引用，或者在组件的整个生命周期内存储可变数据，而不会引发组件重新渲染。

在 React 中，useRef 将 DOM 元素保存为引用，通过 ref.current 获取

基本语法 ​

jsx
const refContainer = useRef(initialValue);
  • initialValue : 可选，初始化时传入的值。
  • refContainer : 返回一个包含 .current 属性的对象，.current 用于存储引用的值。

主要用途 ​

1. 访问 DOM 元素 ​

在 React 中，直接操作 DOM 是不推荐的，但某些场景下是必要的，例如手动设置焦点。

document.getElementById("input").focus(); // 原生 JS 中的用法

在 React 中，useRef 将 DOM 元素保存为引用，通过 ref.current 获取，然后调用其原生方法 focus()。

jsx
import React, { useRef } from "react";

function FocusInput() {
  const inputRef = useRef(null);

  const handleFocus = () => {
    inputRef.current.focus(); // 直接操作 DOM 元素
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="输入内容" />
      <button onClick={handleFocus}>聚焦到输入框</button>
    </div>
  );
}

export default FocusInput;
  • 解释 ：inputRef 被赋值为 <input> 元素的引用，通过 inputRef.current 可以访问该元素。

2. 保存可变数据 ​

useRef 的另一个常见用途是保存一些可变数据，而不希望这些数据的改变触发组件重新渲染。

jsx
import React, { useRef, useState } from "react";

function Counter() {
  const countRef = useRef(0);
  const [state, setState] = useState(0);

  const increment = () => {
    countRef.current++; // 修改 ref 中的值，不会引发重新渲染
    console.log("countRef:", countRef.current);
  };

  const incrementState = () => {
    setState((prev) => prev + 1); // 通过 state 修改触发重新渲染
  };

  return (
    <div>
      <p>countRef: {countRef.current}</p>
      <p>state: {state}</p>
      <button onClick={increment}>增加 countRef</button>
      <button onClick={incrementState}>增加 state</button>
    </div>
  );
}

export default Counter;
  • 解释 ：
  • countRef.current 的值可以随意修改，但不会触发组件重新渲染。
  • state 的修改会引发组件重新渲染。

3. 保存组件实例 ​

在函数组件中无法直接使用 this，但可以通过 useRef 来存储组件实例或某些全局的逻辑数据。

jsx
import React, { useRef, useEffect } from "react";

function Timer() {
  const timerRef = useRef(null);

  useEffect(() => {
    timerRef.current = setInterval(() => {
      console.log("计时中...");
    }, 1000);

    return () => {
      clearInterval(timerRef.current); // 清除定时器
    };
  }, []);

  return <h1>计时器已启动，查看控制台</h1>;
}

export default Timer;
  • 解释 ：
  • timerRef 存储的是一个定时器的 ID。
  • 在 useEffect 的清理函数中，可以通过 timerRef.current 清除定时器。

4. 防止闭包陷阱 ​

在异步操作中，可能会遇到闭包问题。useRef 可以用来存储最新的值，避免因闭包导致的数据不一致。

jsx
import React, { useState, useRef, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);
  const countRef = useRef(count);

  useEffect(() => {
    countRef.current = count; // 更新最新的 count 值
  }, [count]);

  const handleClick = () => {
    setTimeout(() => {
      console.log("最新 count:", countRef.current); // 始终获取最新的值
    }, 3000);
  };

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount((prev) => prev + 1)}>增加</button>
      <button onClick={handleClick}>打印最新 count</button>
    </div>
  );
}

export default Counter;
  • 解释 ：
  • 如果直接使用 count，可能会因为闭包导致拿到的是旧值。
  • 使用 countRef，每次更新时同步 current，可以保证拿到最新的值。

useRef 与 state 的区别 ​

特性useRefstate
重新渲染修改 current不会引发重新渲染修改 state会触发组件重新渲染
初始值设置可以设置任意初始值通过 useState(initialValue)设置初值
适用场景保存 DOM 引用、存储非渲染相关数据存储与 UI 相关的数据

总结 ​

useRef 是一个灵活的工具，主要用来：

  1. 引用 DOM 元素或组件实例。
  2. 保存可变数据（不会引发重新渲染）。
  3. 处理定时器、订阅、或闭包陷阱等场景。

可以将 useRef 看作是 React 提供的一个“静态变量”，它在组件生命周期内保持不变，不会受渲染影响，但却可以动态更新和读取。
